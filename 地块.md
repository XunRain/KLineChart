## 玩家地块模型制作说明

### 1. 目标体验

玩家地块要表现为“自己的小花园”：入口清楚、可种植区域清楚、扩地后土地向后延伸、栅栏跟随变大，并能显示玩家头像和名字。

------

### 2. 每个玩家地块的固定层级

每个地块都放在：

```
Workspace
└─ Gardens
   ├─ Plot1
   ├─ Plot2
   ├─ ...
   └─ Plot8
```

每个 `PlotX` 必须是 `Model`，名字保持 `Plot1`、`Plot2` 这种格式。

建议结构：

```
Plot1 [Model]
├─ SpawnPoint [Part]
├─ PlotSizeReference [Part]
├─ Visual [Folder 或 Model]
│  ├─ PRIM [Part]
│  ├─ PlantAreaColumn1 [Part]
│  ├─ PlantAreaColumn2 [Part]
└─ Signs [Folder]
   ├─ Garden [Model]
   │  └─ CorePart [Part]
   │     └─ SurfaceGui
   └─ Expand [Model]
      └─ CorePart [Part]
         └─ SurfaceGui 或 BillboardGui
```

------

### 3. 必须保留的对象说明

| 路径                      | 类型           | 用途                           | 要求                                        |
| ------------------------- | -------------- | ------------------------------ | ------------------------------------------- |
| `PlotX.SpawnPoint`        | `BasePart`     | 玩家进入/传送到自己花园的位置  | 可透明；必须 Anchored                       |
| `PlotX.PlotSizeReference` | `BasePart`     | 扩地尺寸基准、地块视觉重建基准 | 可透明                                      |
| `PlotX.Visual`            | `Folder/Model` | 地块视觉和种植区域容器         | 可放装饰                                    |
| `Visual.PRIM`             | `BasePart`     | 花园上方 Billboard 归属牌挂点  | 可透明；位置决定头顶牌显示中心              |
| `Visual.PlantAreaColumn1` | `BasePart`     | 第一列可种植区域               | 建议透明；`CanQuery=true`，用于鼠标射线种植 |
| `Visual.PlantAreaColumn2` | `BasePart`     | 第二列可种植区域               | 同上                                        |
| `Signs.Garden`            | `Model`        | 地块主人头像和名字牌           | 需要玩家头像框和名字文字区域                |
| `Signs.Expand`            | `Model`        | 扩地购买牌                     | 全部设置为透明即可                          |

------

### 4. 种植区域要求

`PlantAreaColumn1` 和 `PlantAreaColumn2` 是真正决定能不能种植物的区域。

要求：

```
Visual
├─ PlantAreaColumn1 [Part]
└─ PlantAreaColumn2 [Part]
```

每个 Part 建议属性：

- `Anchored = true`
- `Transparency = 1`
- `CanCollide = false`
- `CanTouch = false`
- `CanQuery = true`
- 顶面高度要贴合视觉泥土表面
- X/Z 尺寸覆盖玩家实际能点击种植的范围

------

### 5. 地块视觉模板资源

运行时地块床面和栅栏会从这里读取：

```
ReplicatedStorage
└─ Assets
   └─ PlotAssets
      ├─ BedSection [Model]
      ├─ FRONT_BedSection [Model]
      └─ FenceThemes
         └─ Default
            ├─ FencePole [Model]
            └─ FenceConnectors [Model]
```

#### `BedSection`

普通土地段。

要求：

- `Model`
- 包含 `BottomFace [BasePart]`
- `BottomFace` 用于计算尺寸和贴地
- 模型朝向要和当前模板一致，代码会旋转摆放

#### `FRONT_BedSection`

入口附近的第一段土地。

要求：

- `Model`
- 建议包含 `BottomFace [BasePart]`
- 用于保留花园入口处特殊形状

#### `FencePole`

单个栅栏柱。

要求：

- `Model`
- 建议包含 `BottomFace [BasePart]`
- 尺寸不要为 0
- Pivot 建议在模型中心或底部中心附近

#### `FenceConnectors`

栅栏横梁/连接段。

要求：

- `Model`
- 建议包含 `BottomFace [BasePart]`
- 会在两个柱子之间重复摆放
- 长度要和 `FencePole` 搭配，避免拼接穿模

------

### 6. Garden 主人牌结构

路径：

```
PlotX
└─ Signs
   └─ Garden
      └─ CorePart
         └─ SurfaceGui
```

`SurfaceGui` 内需要至少有：

```
SurfaceGui
├─ Player
│  └─ TextLabel
└─ ImageLabel
```

用途：

- `Player.TextLabel` 显示：`xxx's Garden`
- `ImageLabel` 显示玩家头像
- 空地块显示：`Empty Garden`

注意：文字必须预留英文长度，比如 `VeryLongPlayerName's Garden`。

------

### 7. Expand 扩地牌结构

路径：

```
PlotX
└─ Signs
   └─ Expand
      └─ CorePart
         └─ SurfaceGui 或 BillboardGui
```

UI 内必须能被代码找到：

```
TextButton
TextLabel
```

用途：

- `TextButton`：点击购买下一块地
- `TextLabel`：显示价格，如 `500¢`、`30K¢`、`MAX`

建议：

- 按钮颜色支持绿色可购买、红色不可购买。
- 满级显示 `MAX`。
- 入口附近摆放，不要挡住玩家移动。
